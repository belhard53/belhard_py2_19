IP-адрес — это уникальный числовой идентификатор устройства в 
        сети (как почтовый адрес дома), а порт — это число (от 0 до 65535), 
        которое указывает на конкретное приложение или службу на этом 
        устройстве (как номер квартиры в доме). 
        Вместе они создают полный адрес для доставки данных: 
        IP-адрес:Порт (например, 192.168.1.1:80), 
        чтобы соединение попадало к нужному сервису (веб-серверу, FTP, игре).

---- сокеты -------------------------------------------------
Сокет — программный интерфейс для обмена данными по сети между процессами на разных машинах или в рамках одного компьютера.

Типы сокетов:
TCP (SOCK_STREAM) — потоковый, надёжный, с контролем доставки и порядка байтов
UDP (SOCK_DGRAM) — датаграммный, быстрый, без гарантии доставки

Адресация:
IPv4: 192.168.1.1:8080 (IP + порт)
IPv6: [::1]:8080

localhost = 127.0.0.1

Порты 0-1023 — привилегированные (требуют прав администратора)
Порты 1024-65535 — пользовательские

Этапы TCP-соединения:
Сервер:
    socket() — создание сокета
    bind((ip, порт)) — привязка к адресу
    listen(queue_size) — переход в режим ожидания подключений
    accept() — ожидание и принятие нового соединения (возвращает новый сокет)

Клиент:
    socket() — создание сокета
    connect((ip, порт)) — установление соединения

Обмен данными:
    send(bytes) / sendall(bytes) — отправка данных
    recv(buffer_size) — приём до buffer_size байт (блокируется до получения данных)
    close() — закрытие соединения

Семантика recv():
    Возвращает b'' (пустые байты) при корректном отключении удалённой стороны
    Блокируется при отсутствии данных
    Может вернуть меньше запрошенного количества байтов

Обработка ошибок:
    OSError / ConnectionRefusedError — адрес недоступен
    TimeoutError — превышение времени ожидания
    Address already in use — порт занят предыдущим процессом

Байтовый характер:
    Все операции работают с байтами (bytes)
    str.encode('utf-8') → bytes
    bytes.decode('utf-8') → str

------------------ HTTP -----------------

Модель «Клиент-Сервер»:
    Клиент (User Agent): Браузер, мобильное приложение, скрипт — инициирует 
                         соединение и отправляет запрос.
    Сервер:  Обрабатывает запрос, возвращает ответ (HTML-страницу, картинку, 
                         данные JSON и т.д.).
Особенности HTTP:
    Протокол без состояния (Stateless): Каждый запрос обрабатывается сервером независимо от предыдущих. 
                Для поддержания состояния используются механизмы вроде Cookies и сессий.
    Работа поверх TCP/IP: HTTP использует надежные TCP-соединения (обычно на порт 80 или 443 для HTTPS).



Структура HTTP
    1. Стартовая строка (start line) — используется для описания версии используемого
    протокола и другой информации — вроде запрашиваемого ресурса или кода ответа.
    Как можно понять из названия, ее содержимое занимает ровно одну строчку.
    2. HTTP-заголовки (HTTP Headers) — несколько строчек текста в определенном
    формате (ключ:значение), которые либо уточняют запрос, либо описывают содержимое тела
    сообщения.
    3. Пустая строка, которая сообщает, что все метаданные для конкретного запроса или
    ответа были отправлены.
    4. Опциональное тело сообщения, которое содержит данные, связанные с запросом, либо
    документ (например HTML-страницу), передаваемый в ответе.

запрос:
    GET /item/<int>/ HTTP/1.1
    Content-Type: text/html; charset=utf-8
    Content-Length: 4
    X-Custom-Header: value

    test

ответ:
    HTTP/1.1 200 OK
    Content-Type: text/html; charset=utf-8

    body



Коды состояния
    100 - служебные
    200 - ок
    300 - перенаправление
    400 - ошибка
    500 - ошибка сервера

Методы запроса 
    GET Запрос на передачу ресурса.
    HEAD Запрос на передачу ресурса, но сам ресурс в ответе не передается, только заголовки.
    POST Передача данных на сервер для обработки указанного ресурса.
    PUT Размещение ресурса на сервере (если такой ресурс уже есть на сервере, то он замещается).
    DELETE Удаление ресурса на сервере.
    CONNECT Установка соединение с сервером на основе ресурса.
    OPTIONS Запрос поддерживаемых методов HTTP для ресурса и других параметров коммуникации.
    TRACE Запрос на трассировку сообщения: сервер должен включить в свой ответ 
            исходный запрос, на который он отвечает. Это полезно, когда запрос 
            проходит через промежуточные устройства, которые могут изменить запрос, 
            например, добавить заголовки.


Основные Headers в HTTP запросе
User-Agent - Заголовок используется для описания клиента, который 
            запрашивает ресурс. Он содержит достаточно много информации 
            о пользовательском окружении. Например, может указать, какой 
            браузер используется в качестве клиента, его версию, а также 
            операционную систему, на которой этот клиент работает.
Refer - Используется для указания того, откуда поступил текущий запрос. 
            Например, если вы решите перейти по какой-нибудь ссылке в этой 
            статье, то вероятнее всего к запросу будет добавлен заголовок 
            Refer: https://иммяэтогосайта.ru
Accept - Позволяет указать, какой тип медиафайлов принимает клиент. 
            В данном заголовке могут быть указаны несколько типов, 
            перечисленные через запятую (‘ , ‘). А для указания того, 
            что клиент принимает любые типы, используется следующая 
            последовательность — */*.
Cookie - Данный заголовок может содержать в себе одну или несколько 
            пар «Куки-Значение» в формате cookie=value. 
            Куки представляют собой небольшие фрагменты данных, 
            которые хранятся как на стороне клиента, так и на сервере, 
            и выступают в качестве идентификатора. Куки передаются вместе 
            с запросом для поддержания доступа клиента к ресурсу. 
            Помимо этого, куки могут использоваться и для других целей, 
            таких как хранение пользовательских предпочтений на сайте и 
            отслеживание клиентской сессии. 
            Несколько кук в одном заголовке могут быть перечислены 
            с помощью символа точка с запятой (‘ ; ‘), который  
            используется как разделитель.
Authorization - Используется в качестве еще одного метода идентификации 
            клиента на сервере. После успешной идентификации сервер 
            возвращает токен, уникальный для каждого конкретного клиента. 
            В отличие от куки, данный токен хранится исключительно на стороне 
            клиента и отправляется клиентом только по запросу сервера. 
            Существует несколько типов аутентификации, конкретный метод 
            определяется тем веб-сервером или веб-приложением, к которому 
            клиент обращается за ресурсом.

Content-Type       
            Text: text/plain - Простой текст, text/html - HTML-документ, text/css - CSS-стили, text/javascript - JavaScript код, text/csv - табличные данные CSV, text/xml - XML-документ.
            Application: application/json - Данные JSON, application/xml - XML-данные, application/pdf - PDF-документ, application/zip - ZIP-архив, application/gzip - GZIP-архив, application/octet-stream - бинарные данные, application/x-www-form-urlencoded - данные формы.
            Image: image/jpeg - JPEG изображение, image/png - PNG изображение, image/gif - GIF изображение, image/svg+xml - SVG векторное изображение, image/webp - WebP изображение, image/x-icon - иконка favicon.
            Audio: audio/mpeg - MP3 аудио, audio/wav - WAV аудио, audio/ogg - Ogg Vorbis аудио.
            Video: video/mp4 - MP4 видео, video/mpeg - MPEG видео, video/quicktime - QuickTime видео, video/webm - WebM видео.
            Font: font/woff - WOFF шрифт, font/woff2 - WOFF2 шрифт, font/ttf - TrueType шрифт.
            Multipart: multipart/form-data - данные формы с файлами, multipart/byteranges - частичный контент.
