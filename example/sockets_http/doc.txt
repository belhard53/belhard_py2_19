
основные сетевые протоколы 
    - UDP - без гарантии доставки пакета
    - TCP - гарантия доставки пакета

IP-адрес — это уникальный числовой идентификатор устройства в 
        сети (как почтовый адрес дома), а порт — это число (от 0 до 65535), 
        которое указывает на конкретное приложение или службу на этом 
        устройстве (как номер квартиры в доме). 
        Вместе они создают полный адрес для доставки данных: 
        IP-адрес:Порт (например, 192.168.1.1:80), 
        чтобы соединение попадало к нужному сервису (веб-серверу, FTP, игре). 


DHCP сервер - устройство(ПО) выдающее ip адреса другим устройствам

192.168.0.1:8080 - связка ip-адрес и порт - используется для отправки пакета в сеть

192.168.0.1, 192.168.1.1 - как правило адреса домашних сетей
127.0.0.1 - localhost - локальный адрес - внутренний адрес устройства

MAC-адрес - уникальный адрес сетевой карты - задается на заводе
например - 00:1A:2B:3C:4D:5E



ip v.4 - 0-255.255.255.255
ip v.6 - 3FFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF




---- сокеты -------------------------------------------------
Сокет — программный интерфейс для обмена данными по сети между процессами на разных машинах или в рамках одного компьютера.

Типы сокетов:
TCP (SOCK_STREAM) — потоковый, надёжный, с контролем доставки и порядка байтов
UDP (SOCK_DGRAM) — датаграммный, быстрый, без гарантии доставки

Адресация:
IPv4: 192.168.1.1:8080 (IP + порт)
IPv6: [::1]:8080

localhost = 127.0.0.1

Порты 0-1023 — привилегированные (требуют прав администратора)
Порты 1024-65535 — пользовательские

Этапы TCP-соединения:
Сервер:
    socket() — создание сокета
    bind((ip, порт)) — привязка к адресу
    listen(queue_size) — переход в режим ожидания подключений
    accept() — ожидание и принятие нового соединения (возвращает новый сокет)

Клиент:
    socket() — создание сокета
    connect((ip, порт)) — установление соединения

Обмен данными:
    send(bytes) / sendall(bytes) — отправка данных
    recv(buffer_size) — приём до buffer_size байт (блокируется до получения данных)
    close() — закрытие соединения

Семантика recv():
    Возвращает b'' (пустые байты) при корректном отключении удалённой стороны
    Блокируется при отсутствии данных
    Может вернуть меньше запрошенного количества байтов

Обработка ошибок:
    OSError / ConnectionRefusedError — адрес недоступен
    TimeoutError — превышение времени ожидания
    Address already in use — порт занят предыдущим процессом

Байтовый характер:
    Все операции работают с байтами (bytes)
    str.encode('utf-8') → bytes
    bytes.decode('utf-8') → str
